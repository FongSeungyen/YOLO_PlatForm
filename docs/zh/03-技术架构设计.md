## 2. 架构概述

### 2.1 设计原则

YOLO Platform V2.0 采用**分层架构设计**，遵循以下核心原则：

1. **模块化设计**：各功能模块独立，职责单一，便于维护和扩展
2. **配置驱动**：通过配置文件管理所有参数，无需修改代码即可调整行为
3. **插件化架构**：支持自定义组件接入，扩展系统能力
4. **主题分离**：UI与业务逻辑分离，支持多主题切换和自定义样式
5. **完整日志追踪**：操作日志和系统日志分离，支持审计和问题排查
6. **多任务支持**：统一架构支持五种视觉任务（Classify/Detect/Segment/Pose/OBB）

### 2.2 技术栈总览

| 层级        | 技术组件                    | 版本             | 用途               |
| --------- | ----------------------- | -------------- | ---------------- |
| **GUI框架** | PySide6 (Qt for Python) | 6.8+           | 跨平台现代化图形界面       |
| **深度学习**  | PyTorch                 | 2.6+           | 模型训练和推理引擎        |
| **目标检测**  | Ultralytics YOLO        | 8.x            | YOLO算法官方实现       |

| **可视化**   | Matplotlib + Seaborn    | 3.9+ / 0.13+   | 训练图表和数据可视化       |

#### 3.2.2 核心层 (core/)

**配置管理** (`config.py`):

```python
@dataclass
class Config:
    """主配置类"""
    app: AppConfig              # 应用配置
    data: DataConfig            # 数据集配置
    augmentation: AugConfig     # 数据增强配置
    model: ModelConfig          # 模型配置
    training: TrainingConfig    # 训练配置
    ui: UIConfig                # UI配置
    paths: PathConfig           # 路径配置
```

配置数据结构：

- `AppConfig`: 应用名称、版本、调试模式、日志级别
- `DataConfig`: 支持格式、数据分割比例、最大工作线程数
- `AugConfig`: 几何变换、颜色变换、高级增强策略
- `ModelConfig`: 支持的模型列表、设备选择、混合精度
- `TrainingConfig`: 训练参数、优化器、学习率调度器
- `UIConfig`: 主题、语言、字体、窗口尺寸

## 8. 认证系统设计

### 8.1 认证流程

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 用户输入  │───→│ 密码验证  │───→│ 会话创建  │───→│ 权限检查  │
│ 用户名密码 │    │(Werkzeug)│    │(Session) │    │ (RBAC)   │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
```

### 8.2 安全机制

- **密码哈希**：使用 Werkzeug 的 `generate_password_hash`，默认 pbkdf2:sha256
- **会话管理**：基于文件的会话存储，支持会话超时
- **权限控制**：基于角色的访问控制（RBAC），支持管理员/用户角色
- **登录保护**：失败次数限制，防止暴力破解

## 10. 性能优化

### 10.1 缓存策略

| 缓存类型   | 用途         | 有效期   | 存储位置  |
| ------ | ---------- | ----- | ----- |
| 数据集元数据 | 数据集信息、类别列表 | 持久化   | 文件系统  |
| 图像缩略图  | 快速预览       | LRU策略 | 内存+磁盘 |
| 模型缓存   | 已加载的模型     | 会话期间  | GPU显存 |
| 配置缓存   | 配置对象       | 应用期间  | 内存    |
| 训练检查点  | 断点续训       | 持久化   | 文件系统  |

### 10.2 多线程设计

```
主线程 (GUI)
    │
    ├─→ 数据加载线程池 (4 workers)
    │   - 图像解码
    │   - 数据预处理
    │
    ├─→ 训练监控线程
    │   - 日志更新
    │   - 图表绘制
    │
    ├─→ 模型推理线程 (AI标注)
    │   - 异步推理
    │   - 结果回调
    │
    └─→ 文件IO线程
        - 模型保存
        - 日志写入
```

### 10.3 内存管理

- **图像懒加载**：仅在需要时加载图像到内存
- **数据生成器**：训练时使用生成器而非一次性加载
- **显存优化**：混合精度训练、梯度检查点
- **垃圾回收**：定期清理无用缓存对象

## 12. 总结

YOLO Platform V2.0 采用分层架构设计，具有以下核心特点：

1. **清晰的层次结构**：GUI层、业务层、核心层、基础设施层严格分离
2. **配置驱动设计**：所有参数通过YAML配置管理，无需修改代码
3. **模块化架构**：各功能模块独立，便于维护和扩展
4. **多主题支持**：6种预设主题，支持动态切换
5. **完整日志系统**：操作日志和系统日志分离，支持审计
6. **安全认证**：基于角色的权限控制，密码安全存储
7. **高性能优化**：缓存策略、多线程、内存管理优化
8. **可扩展性**：插件架构支持自定义扩展
9. **多任务支持**：统一架构支持5种视觉任务
10. **多平台部署**：支持单机、服务端、边缘设备部署
