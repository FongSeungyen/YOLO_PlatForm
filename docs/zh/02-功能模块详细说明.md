## 3. 数据集管理模块

### 3.1 模块概述

数据集管理模块提供完整的数据集生命周期管理功能，支持五种任务类型（Classify/Detect/Segment/Pose/OBB）的数据集创建、导入、标注、增强、验证和导出。

### 3.2 功能详细说明

#### 3.2.1 数据集创建

**支持的任务类型：**

| 任务类型                | 数据格式          | 标注方式       | 适用场景         |
| ------------------- | ------------- | ---------- | ------------ |
| **图像分类 (Classify)** | 文件夹分类         | 无需标注或图像级标签 | 图像分类、内容识别    |
| **目标检测 (Detect)**   | YOLO/COCO/VOC | 矩形边界框      | 物体定位、计数      |
| **实例分割 (Segment)**  | YOLO/COCO     | 多边形/掩码     | 精确轮廓、像素级分析   |
| **姿态估计 (Pose)**     | YOLO/COCO     | 关键点坐标      | 人体/动物姿态分析    |
| **定向检测 (OBB)**      | YOLO          | 旋转矩形框      | 遥感、文本、任意角度目标 |

**创建流程：**

```
选择任务类型 → 配置类别信息 → 设置关键点（Pose）→ 创建数据集 → 添加图像
```

**关键点配置（姿态估计任务）：**

- 支持自定义关键点数量和名称
- 可配置关键点连接关系（骨架）
- 支持多个关键点组（如人体+手部）

#### 3.2.2 数据导入

**支持的格式（按任务类型）：**

**通用格式（所有任务）：**

| 格式     | 说明                 | 文件结构                          |
| ------ | ------------------ | ----------------------------- |
| YOLO格式 | 每张图像对应一个txt文件      | `images/`, `labels/`          |
| COCO格式 | JSON格式，通用标准        | `images/`, `annotations.json` |
| VOC格式  | XML格式，Pascal VOC标准 | `JPEGImages/`, `Annotations/` |

**导入配置选项：**

- 自动识别类别名称
- 训练/验证/测试集比例自动划分
- 类别映射和冲突处理
- 图像格式自动转换

#### 3.2.3 数据增强

**通用增强策略：**

| 增强类型       | 参数    | 说明           | 适用任务                    |
| ---------- | ----- | ------------ | ----------------------- |
| 随机翻转       | 水平/垂直 | 随机水平或垂直翻转图像  | 全任务                     |
| 随机旋转       | 角度范围  | 随机旋转图像       | 全任务                     |
| 随机缩放       | 缩放范围  | 随机缩放图像尺寸     | 全任务                     |
| 随机裁剪       | 裁剪比例  | 随机裁剪图像区域     | 全任务                     |
| 颜色抖动       | HSV调整 | 调整亮度/对比度/饱和度 | 全任务                     |
| Mosaic     | 网格大小  | 将4张图像拼接成一张   | Detect/Segment/Pose/OBB |
| MixUp      | 混合比例  | 图像混合增强       | Detect/Segment          |
| Copy-Paste | 复制粘贴  | 复制对象到其他图像    | Detect/Segment          |

**任务特定增强：**

**姿态估计专用：**

- 关键点抖动：对关键点坐标添加微小扰动
- 遮挡模拟：随机隐藏部分关键点
- 翻转关键点映射：左右关键点对称变换

**OBB专用：**

- 旋转角度保持：确保旋转框角度一致性
- 长宽比保持：维持定向框的几何特性

#### 3.2.4 数据验证

**验证项目：**

- 图像文件完整性检查（损坏检测）
- 标注文件格式验证
- 坐标范围检查（是否越界）
- 类别ID一致性检查
- 关键点数量验证（Pose任务）
- 多边形闭合性检查（Segment任务）
- 旋转框有效性检查（OBB任务）

## 4. 智能标注模块

### 4.1 模块概述

智能标注模块提供AI辅助标注功能，支持全五种任务类型的交互式标注，大幅提升标注效率。

### 4.2 功能详细说明

#### 4.2.1 AI辅助标注

**工作流程：**

```
选择模型 → 配置推理参数 → AI自动标注 → 人工审核修正 → 保存标注
```

**支持的模型类型：**

| 任务类型     | 推荐模型                             | 模型来源  |
| -------- | -------------------------------- | ----- |
| Classify | yolo11n-cls.pt, yolo26n-cls.pt   | 官方预训练 |
| Detect   | yolo11n.pt, yolo26n.pt           | 官方预训练 |
| Segment  | yolo11n-seg.pt, yolo26n-seg.pt   | 官方预训练 |
| Pose     | yolo11n-pose.pt, yolo26n-pose.pt | 官方预训练 |
| OBB      | yolo11n-obb.pt, yolo26n-obb.pt   | 官方预训练 |

**AI标注参数配置：**

**通用参数：**

- 置信度阈值：过滤低置信度检测结果（默认0.25）
- IoU阈值：NMS去重阈值（默认0.45）
- 最大检测数：单图最大检测数量（默认300）
- 设备选择：CPU/GPU自动或手动指定

**任务特定参数：**

**姿态估计 (Pose)：**

- 关键点置信度阈值：控制关键点可见性判断
- 骨架连接显示：是否显示关键点连接线
- 关键点名称标签：显示关键点名称

**实例分割 (Segment)：**

- 最大轮廓点数：控制分割精度（默认500）
- 轮廓平滑度：轮廓简化程度

**OBB：**

- 角度精度：旋转框角度计算精度
- 长宽比限制：过滤异常比例的框

#### 4.2.2 交互式标注工具

**标注工具类型（按任务）：**

| 任务           | 标注工具  | 操作方式        |
| ------------ | ----- | ----------- |
| **Classify** | 类别选择器 | 下拉框/快捷键选择类别 |
| **Detect**   | 矩形框工具 | 拖拽绘制边界框     |
| **Segment**  | 多边形工具 | 点击创建顶点，闭合完成 |
| **Pose**     | 关键点工具 | 点击放置关键点     |
| **OBB**      | 旋转框工具 | 绘制矩形+旋转调整   |

**通用编辑功能：**

- 选择/移动/删除标注
- 撤销/重做（Ctrl+Z/Ctrl+Y）
- 复制/粘贴标注
- 快捷键支持
- 批量标注模式

#### 4.2.3 标注审核流程

**标注状态标记：**

-  已标注
-  未标注

**AI标记管理：**

- AI生成的标注显示特殊标记（🤖）
- 修改后自动清除AI标记
- 支持按AI标记筛选图像

**质量检查：**

- 标注完整性检查
- 类别一致性检查
- 重叠检测（Detect/Segment）
- 关键点可见性检查（Pose）

### 4.3 标注工作流

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│   选择图像    │───→│   AI预标注    │───→│   人工审核    │
└──────────────┘    └──────────────┘    └──────────────┘
                                               │
                                               ▼
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│   下一张图像  │←───│   保存标注    │←───│   修正错误    │
└──────────────┘    └──────────────┘    └──────────────┘
```

### 5.5 断点续训

**自动保存策略：**

- 每 10 个 epoch 自动保存检查点

**恢复训练方式：**

1. 自动检测未完成的训练任务
2. 手动选择检查点文件
3. 从最佳模型继续微调

## 6. 模型评估模块

### 6.1 模块概述

模型评估模块提供全面的模型性能评估功能，支持五种任务类型的评估指标计算和可视化分析。

### 6.2 评估指标

#### 6.2.1 通用指标

| 指标        | 说明   | 计算公式                    | 适用任务               |
| --------- | ---- | ----------------------- | ------------------ |
| Precision | 精确率  | TP / (TP + FP)          | Detect/Segment/OBB |
| Recall    | 召回率  | TP / (TP + FN)          | Detect/Segment/OBB |
| F1-Score  | F1分数 | 2 \* (P \* R) / (P + R) | Detect/Segment/OBB |

#### 6.2.2 任务特定指标

**图像分类 (Classify)：**

| 指标                 | 说明            |
| ------------------ | ------------- |
| Top-1 Accuracy     | 最高置信度类别正确率    |
| Top-5 Accuracy     | 前5个置信度类别包含正确率 |
| Per-class Accuracy | 各类别单独准确率      |
| Confusion Matrix   | 混淆矩阵          |

**目标检测/分割/OBB (Detect/Segment/OBB)：**

| 指标            | 说明                |
| ------------- | ----------------- |
| mAP\@0.5      | IoU=0.5时的平均精度     |
| mAP\@0.5:0.95 | IoU从0.5到0.95的平均精度 |
| AP50          | 单个IoU阈值下的精度       |
| APs/APm/APl   | 小/中/大目标AP         |
| FPS           | 每秒处理帧数            |

**姿态估计 (Pose)：**

| 指标                                    | 说明       |
| ------------------------------------- | -------- |
| mAPpose                               | 姿态估计平均精度 |
| OKS (Object Keypoint Similarity)      | 关键点相似度   |
| PCK (Percentage of Correct Keypoints) | 正确关键点百分比 |

### 6.3 可视化分析

**支持的图表类型：**

- 混淆矩阵热力图
- PR曲线（Precision-Recall）
- F1曲线
- 损失曲线
- 各类别AP柱状图
- 预测结果可视化

### 7.1 模块概述

模型部署模块提供多种部署方式，支持将训练好的模型部署到云端服务器、边缘设备和移动端应用。

### 7.2 API服务部署

**API端点：**

| 端点              | 方法   | 说明    | 参数            |
| --------------- | ---- | ----- | ------------- |
| /predict        | POST | 单图像推理 | file: 图像文件    |
| /predict/batch  | POST | 批量推理  | files: 图像文件列表 |
| /predict/stream | POST | 流式推理  | file: 视频流     |
| /health         | GET  | 健康检查  | 无             |
| /info           | GET  | 模型信息  | 无             |
| /metrics        | GET  | 性能指标  | 无             |

**请求示例：**

```bash
# 单图像推理
curl -X POST "http://localhost:8000/predict" \
  -H "Content-Type: multipart/form-data" \
  -F "file=@image.jpg"

# 批量推理
curl -X POST "http://localhost:8000/predict/batch" \
  -H "Content-Type: multipart/form-data" \
  -F "files=@image1.jpg" \
  -F "files=@image2.jpg"
```

### 7.3 模型优化

**优化技术对比：**

| 技术        | 模型大小 | 推理速度   | 精度损失 | 适用平台       |
| --------- | ---- | ------ | ---- | ---------- |
| FP32（原始）  | 100% | 1x     | 0%   | 通用         |
| FP16（半精度） | 50%  | 1.5-2x | <1%  | NVIDIA GPU |
| INT8（量化）  | 25%  | 2-4x   | 2-5% | 支持INT8设备   |
| 动态量化      | 25%  | 1.5-2x | <2%  | CPU        |
| 剪枝（30%）   | 70%  | 1.2x   | <3%  | 通用         |

**支持的导出格式：**

| 格式              | 扩展名       | 适用平台         | 优化支持          |
| --------------- | --------- | ------------ | ------------- |
| PyTorch         | .pt       | Python环境     | 基础优化          |
| ONNX            | .onnx     | 跨平台          | FP16/INT8     |
| TensorRT        | .engine   | NVIDIA GPU   | FP16/INT8/层融合 |
| TensorFlow Lite | .tflite   | 移动端/嵌入式      | INT8/量化       |
| OpenVINO        | .xml/.bin | Intel设备      | FP16/INT8     |
| CoreML          | .mlmodel  | Apple设备      | FP16          |
| Paddle          | .pdmodel  | PaddlePaddle | FP16          |

## 10. 总结

YOLO Platform V2.0 通过六大核心模块的协同工作，为用户提供了一站式的计算机视觉解决方案：

1. **数据集管理模块**：支持5种任务类型，10+种数据格式，完整的数据生命周期管理
2. **智能标注模块**：AI辅助标注，全任务支持，大幅提升标注效率
3. **模型训练模块**：支持YOLOv8/YOLO11/YOLO26全系列，丰富的训练配置和监控
4. **模型评估模块**：全面的评估指标，多维度可视化分析
5. **模型部署模块**：API服务、边缘部署、多格式导出全覆盖
6. **系统工具模块**：系统诊断、缓存清理、日志管理等维护工具

各模块之间紧密协作，形成完整的工作流，帮助用户快速构建和部署高性能的计算机视觉应用。
