## 2. 安装相关问题

### Q1: 安装时提示 "pip is not recognized"

**问题描述：** 命令行中运行 pip 命令时提示找不到命令。

**解决方案：**

**Windows：**
1. 重新安装 Python，安装时勾选 **"Add Python to PATH"**
2. 或手动添加 Python 和 Scripts 目录到系统 PATH

**Linux/macOS：**
```bash
# 使用包管理器安装 pip
sudo apt install python3-pip  # Ubuntu/Debian
brew install python@3.10      # macOS
```

---

### Q2: CUDA版本不匹配错误

**问题描述：** 运行时提示 `CUDA error: no kernel image is available for execution on the device`

**解决方案：**

1. 检查显卡支持的 CUDA 版本：
   ```bash
   nvidia-smi
   ```

2. 卸载现有 PyTorch：
   ```bash
   pip uninstall torch torchvision torchaudio
   ```

3. 安装对应 CUDA 版本的 PyTorch：
   ```bash
   # CUDA 12.4
   pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu124

   # CUDA 11.8
   pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu118
   ```

---

### Q3: pip安装依赖时超时

**问题描述：** 安装依赖时连接超时或速度极慢。

**解决方案：**

使用国内镜像源：
```bash
# 清华源
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 阿里云
pip install -r requirements.txt -i https://mirrors.aliyun.com/pypi/simple/

# 豆瓣源
pip install -r requirements.txt -i http://pypi.douban.com/simple/
```

永久配置：
```bash
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
```

## 3. 运行相关问题

### Q4: 启动时提示缺少DLL文件

**问题描述：** Windows 系统启动时提示缺少 `VCRUNTIME140.dll` 或类似错误。

**解决方案：**

1. 下载并安装 Visual C++ Redistributable：
   - [下载链接](https://aka.ms/vs/17/release/vc_redist.x64.exe)

2. 或安装完整版 Visual Studio Community

---

### Q5: 界面显示异常/错位

**问题描述：** 窗口显示不完整、文字模糊或布局错乱。

**解决方案：**

1. **调整显示缩放：**
   - Windows：设置 → 系统 → 显示 → 缩放与布局 → 设置为 100%

2. **更新显卡驱动：**
   - NVIDIA: [GeForce Experience](https://www.nvidia.com/geforce/geforce-experience/)
   - AMD: [Radeon Software](https://www.amd.com/support)
   - Intel: [Intel Driver & Support Assistant](https://www.intel.com/content/www/us/en/download-center/home.html)

3. **切换主题：**
   - 应用内点击主题切换按钮（深色/浅色）

---

### Q6: 模型下载失败/超时

**问题描述：** 首次使用 AI 标注或训练时模型下载失败。

**解决方案：**

1. **检查网络连接**

2. **配置代理（如需）：**
   ```bash
   # 设置环境变量
   export HTTP_PROXY=http://proxy.example.com:8080
   export HTTPS_PROXY=http://proxy.example.com:8080
   ```

3. **手动下载模型：**
   ```bash
   # 创建目录
   mkdir -p workspace/models/pretrained
   cd workspace/models/pretrained

   # 下载预训练模型
   wget https://github.com/ultralytics/assets/releases/download/v8.3.0/yolo11n.pt
   wget https://github.com/ultralytics/assets/releases/download/v8.3.0/yolo11n-cls.pt
   ```

## 4. 数据集相关问题

### Q7: 导入数据集格式错误

**问题描述：** 导入 COCO/YOLO 格式数据时提示格式错误。

**解决方案：**

**COCO 格式检查：**
```json
{
  "images": [
    {"id": 1, "file_name": "image1.jpg", "height": 480, "width": 640}
  ],
  "annotations": [
    {"id": 1, "image_id": 1, "category_id": 1, "bbox": [x, y, w, h]}
  ],
  "categories": [
    {"id": 1, "name": "person"}
  ]
}
```

**YOLO 格式检查：**
- 图像：`images/train/image1.jpg`
- 标注：`labels/train/image1.txt`
- 标注文件格式：`class_id x_center y_center width height`（归一化坐标）

**类别文件：** 确保 `classes.txt` 或 `dataset.yaml` 存在且正确

---

### Q8: 数据集验证失败

**问题描述：** 数据集验证时提示图像损坏或标注错误。

**解决方案：**

1. **检查损坏图像：**
   ```python
   from PIL import Image
   import os

   for img_path in image_list:
       try:
           img = Image.open(img_path)
           img.verify()
       except Exception as e:
           print(f"损坏图像: {img_path}")
           os.remove(img_path)  # 删除或修复
   ```

2. **检查标注越界：**
   - 确保坐标值在 [0, 1] 范围内
   - 检查宽高不为负数

3. **使用工具修复：**
   - 应用内的"数据验证"功能会生成修复建议

---

### Q9: 姿态估计关键点配置错误

**问题描述：** Pose 任务提示关键点数量不匹配。

**解决方案：**

1. **检查配置文件：**
   ```yaml
   # data.yaml
   path: ./dataset
   train: images/train
   val: images/val

   kpt_shape: [17, 3]  # [关键点数, 每个关键点维度]
   flip_idx: [0, 2, 1, 4, 3, 6, 5, 8, 7, 10, 9, 12, 11, 14, 13, 16, 15]

   names:
     0: person
   ```

2. **关键点名称：**
   - 在创建数据集时正确定义关键点名称列表
   - 确保标注文件中的关键点顺序与配置一致

## 5. 训练相关问题

### Q10: 训练时显存不足 (CUDA out of memory)

**问题描述：** 训练过程中提示显存溢出。

**解决方案：**

1. **减小批次大小：**
   ```yaml
   batch: 8  # 从16减小到8或4
   ```

2. **减小图像尺寸：**
   ```yaml
   imgsz: 480  # 从640减小到480或320
   ```

3. **使用更小的模型：**
   - yolo11n (最小) → yolo11s → yolo11m → yolo11l → yolo11x (最大)

4. **启用混合精度：**
   ```yaml
   half: true  # 在训练配置中启用
   ```

5. **清空显存：**
   ```python
   import torch
   torch.cuda.empty_cache()
   ```

---

### Q11: 训练损失不下降

**问题描述：** 训练多轮后损失值没有明显下降。

**解决方案：**

1. **检查学习率：**
   - 学习率过大：损失震荡不收敛
   - 学习率过小：收敛速度极慢
   - 建议：使用学习率预热和调度器

2. **检查数据质量：**
   - 确保标注正确
   - 检查类别不平衡问题
   - 验证数据增强策略

3. **调整优化器：**
   ```yaml
   optimizer: AdamW  # 尝试不同的优化器
   lr0: 0.001       # 调整初始学习率
   ```

4. **增加数据量：**
   - 每个类别至少需要 100+ 张图像
   - 使用数据增强扩充数据集

---

### Q12: 训练速度很慢

**问题描述：** 训练速度远低于预期。

**解决方案：**

1. **使用 GPU 训练：**
   ```yaml
   device: 0  # 使用GPU 0
   ```

2. **调整工作线程数：**
   ```yaml
   workers: 8  # 根据CPU核心数调整
   ```

3. **关闭不必要的日志：**
   ```yaml
   verbose: false
   ```

4. **使用缓存：**
   ```yaml
   cache: true  # 缓存数据集到内存
   ```

## 6. AI标注相关问题

### Q13: AI标注结果不准确

**问题描述：** AI辅助标注的结果与预期差距较大。

**解决方案：**

1. **调整置信度阈值：**
   - 降低阈值：召回更多目标（可能增加误检）
   - 提高阈值：减少误检（可能漏检）
   - 建议：从 0.25 开始调整

2. **使用合适的模型：**
   - 通用场景：yolo11n.pt
   - 特定领域：使用自定义训练的模型

3. **预处理图像：**
   - 确保图像清晰度
   - 调整亮度和对比度

---

### Q14: AI标注时程序崩溃

**问题描述：** 点击 AI 标注后程序无响应或崩溃。

**解决方案：**

1. **检查模型文件：**
   - 确保模型文件完整未损坏
   - 重新下载模型文件

2. **降低图像分辨率：**
   - 大分辨率图像可能导致内存溢出
   - 先在较小图像上测试

3. **查看日志：**
   - 检查日志文件获取详细错误信息
   - 路径：`logs/app_YYYYMMDD.log`

## 7. 模型部署相关问题

### Q15: 导出ONNX模型失败

**问题描述：** 导出 ONNX 格式时提示错误。

**解决方案：**

1. **安装ONNX依赖：**
   ```bash
   pip install onnx onnxruntime
   ```

2. **检查模型路径：**
   - 确保模型文件存在且未损坏

3. **简化导出参数：**
   ```python
   model.export(format='onnx', simplify=True)
   ```

---

### Q16: API服务启动失败

**问题描述：** 启动 API 服务时端口被占用。

**解决方案：**

1. **更换端口：**
   ```bash
   python -m uvicorn main:app --port 8001
   ```

2. **查找并关闭占用端口的进程：**

   **Windows：**
   ```cmd
   netstat -ano | findstr :8000
   taskkill /PID <PID> /F
   ```

   **Linux/macOS：**
   ```bash
   lsof -i :8000
   kill -9 <PID>
   ```
