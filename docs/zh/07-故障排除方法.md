## 2. 故障排除流程

### 2.1 标准排障步骤

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   发现问题   │───→│   收集信息   │───→│   定位原因   │───→│   解决问题   │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
       │                  │                  │                  │
       │                  │                  │                  │
       ▼                  ▼                  ▼                  ▼
  观察现象          查看日志          分析错误          验证修复
  复现问题          检查配置          搜索文档          回归测试
```

遇到问题时，请收集以下信息：

1. **环境信息**
   - 操作系统版本

   - CUDA 版本 (`nvidia-smi`)

   - 完整的错误消息

3. **复现步骤**
   - 具体操作流程
   - 输入数据样本
   - 配置参数设置

## 3. 启动故障

### 3.1 应用无法启动

#### 症状
双击运行或执行 `python main.py` 后无反应或立即退出。

#### 排查步骤

**Step 1: 检查Python环境**
```bash
python --version  # 应为 3.10.x
pip --version
```

**Step 2: 检查依赖安装**
```bash
pip list | findstr torch      # Windows
pip list | grep torch         # Linux/macOS
```

**Step 3: 查看错误日志**
```bash
# Windows
type logs\app_*.log

# Linux/macOS
cat logs/app_*.log
```

**Step 4: 测试最小启动**
```python
python -c "from PySide6.QtWidgets import QApplication; print('PySide6 OK')"
python -c "import torch; print('PyTorch:', torch.__version__)"
```

#### 常见原因与解决

| 原因 | 解决 |
|------|------|
| Python版本不兼容 | 安装Python 3.10-3.12 |
| 依赖缺失 | 重新运行 `pip install -r requirements.txt` |
| 配置文件损坏 | 删除 `config/config.yaml` 重新生成 |
| 权限不足 | 以管理员身份运行 |

### 3.2 界面显示异常

#### 症状
窗口显示不完整、文字模糊、布局错乱或黑屏。

#### 排查步骤

**Step 1: 检查显卡驱动**
```bash
# Windows - 设备管理器
# Linux
lspci | grep VGA
nvidia-smi

# macOS
system_profiler SPDisplaysDataType
```

**Step 2: 测试不同主题**
```python
# 编辑 config/config.yaml
ui:
  theme: "light"  # 切换到浅色主题
```

**Step 3: 禁用硬件加速**
```bash
# Windows环境变量
set QT_OPENGL=software
python main.py
```

**Step 4: 检查显示缩放**
- Windows: 设置 → 系统 → 显示 → 缩放与布局 → 100%
- Linux: `xrandr --output <display> --scale 1x1`

## 4. 数据集故障

### 4.1 数据集导入失败

#### 症状
导入数据集时提示格式错误或导入中断。

#### 排查步骤

**Step 1: 检查文件结构**
```
# YOLO格式检查
dataset/
├── images/
│   ├── train/
│   └── val/
└── labels/
    ├── train/
    └── val/
```

**Step 2: 验证标注文件**
```bash
# 检查标注格式
head -n 5 labels/train/image1.txt
# 应为: class_id x_center y_center width height (归一化值)
```

**Step 3: 检查图像完整性**
```python
from PIL import Image
import os

for root, dirs, files in os.walk('dataset/images'):
    for f in files:
        path = os.path.join(root, f)
        try:
            img = Image.open(path)
            img.verify()
        except Exception as e:
            print(f"损坏图像: {path} - {e}")
```

**Step 4: 检查类别映射**
```bash
# 确保classes.txt存在
 cat dataset/classes.txt
```

---

### 4.2 标注保存失败

#### 症状
标注后点击保存无反应或提示保存失败。

#### 排查步骤

**Step 1: 检查目录权限**
```bash
# Windows
icacls dataset\labels

# Linux/macOS
ls -la dataset/labels
```

**Step 2: 检查磁盘空间**
```bash
# Windows
dir /q

# Linux/macOS
df -h
```

**Step 3: 检查标注格式**
- 确保坐标值在 [0, 1] 范围内
- 检查是否有非法字符

**Step 4: 查看日志**
```bash
grep -i "save\|error" logs/app_*.log
```

## 5. 训练故障

### 5.1 训练无法启动

#### 症状
点击开始训练后无反应或立即报错。

#### 排查步骤

**Step 1: 检查数据集配置**
```yaml
# data.yaml
path: ./dataset  # 路径是否正确
train: images/train
val: images/val

names:
  0: class1
  1: class2
```

**Step 2: 检查GPU可用性**
```python
import torch
print(torch.cuda.is_available())
print(torch.cuda.device_count())
print(torch.cuda.get_device_name(0))
```

**Step 3: 检查模型文件**
```bash
ls -lh workspace/models/pretrained/yolo11n.pt
```

**Step 4: 测试最小训练**
```python
from ultralytics import YOLO
model = YOLO('yolo11n.pt')
model.train(data='dataset/data.yaml', epochs=1, imgsz=320)
```

---

### 5.2 训练中断/崩溃

#### 症状
训练过程中程序崩溃或系统死机。

#### 排查步骤

**Step 1: 检查显存使用**
```bash
nvidia-smi -l 1  # 每秒刷新
```

**Step 2: 降低资源占用**
```yaml
# 修改训练配置
batch: 4        # 减小批次
imgsz: 480      # 减小图像尺寸
workers: 2      # 减少数据加载线程
```

**Step 3: 检查系统日志**
```bash
# Windows - 事件查看器
eventvwr.msc

# Linux
dmesg | tail -50
journalctl -xe
```

**Step 4: 启用断点续训**
```python
# 自动保存检查点
model.train(resume=True)  # 从最后检查点恢复
```

---

### 5.3 训练结果异常

#### 症状
损失值不下降、mAP为0或训练速度极慢。

#### 排查步骤

**Step 1: 检查学习率**
```python
# 学习率过大：损失震荡
# 学习率过小：收敛缓慢
# 建议范围
lr0: 0.001  # 较小数据集
lr0: 0.01   # 大数据集
```

**Step 2: 检查数据增强**
```yaml
# 过度增强会导致问题
augmentation:
  hsv_h: 0.015  # 适中
  hsv_s: 0.7
  hsv_v: 0.4
  degrees: 0.0  # 如果不需要旋转
```

**Step 3: 验证标注质量**
- 可视化检查标注是否正确
- 确保类别ID连续从0开始
- 检查是否有空标注文件

**Step 4: 检查类别平衡**
```python
# 统计各类别数量
import yaml
from collections import Counter

with open('data.yaml') as f:
    data = yaml.safe_load(f)

# 分析标注分布
```

## 6. 推理与部署故障

### 6.1 模型推理失败

#### 症状
加载模型后推理报错或无结果。

#### 排查步骤

**Step 1: 检查模型文件**
```python
from ultralytics import YOLO

model = YOLO('path/to/model.pt')
print(model.info())  # 查看模型信息
```

**Step 2: 检查输入图像**
```python
from PIL import Image

img = Image.open('test.jpg')
print(img.size, img.mode)  # 检查尺寸和模式

# 转换为RGB
if img.mode != 'RGB':
    img = img.convert('RGB')
```

**Step 3: 测试最小推理**
```python
import torch
from ultralytics import YOLO

model = YOLO('yolo11n.pt')
results = model('test.jpg')
print(results[0].boxes)
```

**Step 4: 检查设备选择**
```python
# 强制使用CPU测试
model = YOLO('model.pt')
results = model('test.jpg', device='cpu')
```

```bash

python -m uvicorn main:app --log-level debug
```

## 7. 系统工具故障

### 7.1 系统诊断无法打开

#### 症状
点击系统诊断时提示语法错误。

#### 原因与解决

**原因：** [diagnostic_dialog.py:461](file:///d:\yolo_platform_V2.0\gui\dialogs\diagnostic_dialog.py#L461) 行字符串引号嵌套错误

**解决：** 已修复，将中文引号改为英文引号：
```python
# 修复前
self.status_label = QLabel("点击"开始诊断"按钮开始系统检查")

# 修复后
self.status_label = QLabel('点击"开始诊断"按钮开始系统检查')
```

---

### 7.2 缓存清理异常退出

#### 症状
运行缓存清理扫描时程序崩溃。

#### 原因与解决

**原因：**
1. `temp_api_server.py` 被错误地放入 `SKIP_DIRS`
2. 文件扫描时缺少异常处理

**解决：** 已修复 [cache_cleanup_dialog.py](file:///d:\yolo_platform_V2.0\gui\dialogs\cache_cleanup_dialog.py)

1. 添加独立的 `SKIP_FILES` 集合
2. 增强异常处理（捕获 PermissionError 等）
3. 添加多层 try-except 保护

---

### 7.3 日志查看器显示异常

#### 症状
日志列表背景颜色交替显示。

#### 原因与解决

**原因：** 默认启用交替行背景色

**解决：** 已修复 [log_viewer_dialog.py](file:///d:\yolo_platform_V2.0\gui\dialogs\log_viewer_dialog.py)

```python
self.log_table.setAlternatingRowColors(False)
self.log_table.setStyleSheet("""
    QTableWidget {
        background-color: transparent;
        alternate-background-color: transparent;
    }
""")
```

## 8. 性能问题

### 8.1 界面卡顿

#### 症状
操作界面时响应缓慢或卡顿。

#### 优化建议

1. **减少数据集预览数量**
   - 在设置中限制同时加载的图像数量

2. **关闭实时预览**
   - 在标注界面关闭自动保存缩略图

3. **增加内存分配**
   ```bash
   # Python启动参数
   python -Xmx4g main.py  # 分配4GB内存
   ```

4. **清理缓存**
   - 使用系统工具的缓存清理功能

---

### 8.2 训练速度慢

#### 症状
训练速度远低于预期。

#### 优化建议

1. **使用GPU训练**
   ```yaml
   device: 0  # 指定GPU
   ```

2. **调整工作线程**
   ```yaml
   workers: 8  # 根据CPU核心数
   ```

3. **启用缓存**
   ```yaml
   cache: true  # 缓存数据到内存
   ```

4. **使用混合精度**
   ```yaml
   amp: true
   ```

5. **优化数据加载**
   - 使用SSD存储数据
   - 减少数据增强复杂度

## 10. 日志分析

### 10.1 日志位置

| 操作系统 | 路径 |
|----------|------|
| Windows | `logs\app_YYYYMMDD.log` |
| Linux | `logs/app_YYYYMMDD.log` |

### 10.2 常用日志查询

```bash
# 查看最近的错误
grep -i "error\|exception" logs/app_*.log | tail -20

# 查看训练相关日志
grep -i "train\|epoch" logs/app_*.log

# 查看特定时间段的日志
awk '/2026-04-22 15:00/,/2026-04-22 16:00/' logs/app_20260422.log

# 统计错误类型
grep -i "error" logs/app_*.log | cut -d':' -f3 | sort | uniq -c | sort -nr
```

### 12.2 故障速查表

| 问题现象 | 可能原因 | 快速解决 |
|----------|----------|----------|
| 无法启动 | 依赖缺失 | `pip install -r requirements.txt` |
| CUDA错误 | 版本不匹配 | 重装对应CUDA版本的PyTorch |
| 显存不足 | 批次过大 | 减小batch size |
| 导入失败 | 格式错误 | 检查文件结构和格式 |
| 保存失败 | 权限不足 | 检查目录权限 |
| 界面卡顿 | 内存不足 | 关闭其他应用/增加内存 |
| 训练中断 | 资源耗尽 | 降低batch/imgsz/workers |
| API无响应 | 端口占用 | 更换端口或关闭占用进程 |
